# 整理交互重构说明

本次整理交互改造的目标是：减少 token 浪费，让对话更自然。

## 核心变化

- `submit_plan_patch` 改为 `submit_plan_diff`
- 待定计划由系统本地维护，模型只提交增量变更
- 默认展示摘要视图，不再首屏直接输出完整计划
- 支持本地快捷意图：`看明细`、`看改动`、`看待确认项`、`执行`
- `Review/` 下的未决项可在执行前按默认策略自动收口

## 当前交互节奏

1. 扫描目录并生成分析结果
2. 模型提交初版整理 diff
3. 系统展示摘要视图
4. 用户继续调整，或查看明细/改动/待确认项
5. 用户输入 `执行` 后进入预检
6. 用户输入 `YES` 后真正执行

## 设计取舍

- 严格校验仍保留在最终计划阶段
- 本地快捷意图只处理低歧义查看/执行动作
- 其余修改请求仍交给模型理解
